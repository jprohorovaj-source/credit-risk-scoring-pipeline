# Разработка ML-пайплайна кредитного скоринга (БКИ)

Репозиторий содержит финальный проект по разработке и оптимизации модели бинарной классификации для оценки кредитных рисков (прогнозирование дефолта заемщика — просрочка 90+ дней).

## Бизнес-задача и Данные
* **Исходные данные:** Анонимизированный транзакционный массив БКИ объемом **26 млн записей** (4.5 ГБ в формате Parquet).
* **Специфика:** Панельная структура данных (несколько записей на одного клиента) и жесткий дисбаланс классов (**3.55% дефолтных заемщиков**, распределение Бернулли).
* **Критерий качества по ТЗ:** Метрика **ROC-AUC >= 0.75**.

## Ключевые инженерные решения

### 1. Вертикальное пакетирование и Оптимизация RAM
Для обхода Memory Overflow в условиях ограниченной памяти сервера (Google Colab, 12 ГБ RAM) реализован кастомный подход предобработки:
* **Даункастинг типов данных:** Принудительное сжатие типов данных с проверкой числовых диапазонов (`int64` -> `int8`, `float64` -> `float32`) с помощью функции `reduce_mem_usage`.
* **Поколоночное чтение:** Чтение признаков строго вертикально (по одной колонке за раз) через `pd.read_parquet(..., columns=[col])`.
* **Агрегация:** Применение One-Hot Encoding внутри пакета с последующей группировкой по ID клиента (.sum()) для формирования частотного профиля заемщика.
* **Результат:** Снижение пиковой нагрузки на RAM в 6 раз (**с 12 ГБ до 2 ГБ**).

### 2. Эксперименты и Моделирование (LightGBM + Optuna)
* **Шаг 1 (Baseline):** Обучение градиентного бустинга LightGBM без балансировки классов. Результат: **ROC-AUC = 0.7505** (подтверждена высокая предсказательная сила частотных признаков).
* **Шаг 2 (Ручные веса):** Попытка балансировки через `scale_pos_weight = 27.18`. Результат: **ROC-AUC рухнул до 0.6874** (алгоритм начал перестраховываться, сломав точность ранжирования).
* **Шаг 3 (Оптимизация):** Автоматизированный подбор мягких весов с помощью фреймворка **Optuna**. Результат за 10 итераций: **ROC-AUC = 0.7607** (целевое ТЗ успешно перевыполнено).

## Деплой и Готовность к внедрению
* **Архитектура:** Весь цикл препроцессинга (кастомный трансформер `CreditRiskFeatureAssembler`) и финальная модель упакованы в единый автономный объект `Pipeline` библиотеки `scikit-learn`.
* **Артефакт:** Полный пайплайн на 418 итоговых признаков экспортирован в бинарный формат `credit_risk_pipeline.pkl`.
* **Docker:** Подготовлен `Dockerfile` (базовый образ `python:3.10-slim`) для контейнеризации приложения и поднятия локального сервера предсказаний.
